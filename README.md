## SLA 的意思，三句话说明白

**SLA**，全称 **Service Level Agreement**，中文叫**服务等级协议**，或者服务级别协议。

简单说：这是服务商和你之间签的一份"军令状"。

它用量化指标明确约定：

- 服务能稳定运行多少时间（可用性 / 正常运行时间）
- 出了问题多久响应、多久解决（响应时间 / 恢复时间）
- 如果没达标，服务商赔什么（补偿机制）

没有 SLA，服务商说"我们很稳定"——那是广告词。有了 SLA，服务商说"99.9% 可用性，否则补偿"——那是法律层面的承诺，白纸黑字。

这就是 SLA 的本质意义：**把口头承诺变成可追究的合同条款**。

## SLA 里最常见的指标详解

### 1. 可用性（Availability）

这是 SLA 最核心的数字，通常表示为百分比。

| 可用性 | 每年最多允许宕机时长 |
|--------|---------------------|
| 99% | 约 3 天 15 小时 |
| 99.5% | 约 1 天 19 小时 |
| 99.9% | 约 8 小时 46 分钟 |
| 99.95% | 约 4 小时 23 分钟 |
| 99.99% | 约 52 分钟 |
| 99.999% | 约 5 分钟 |

注意：这是**允许**宕机的时长，不是说一定会宕机这么久。

### 2. 几个"9"是什么意思

你经常听到"四个9""五个9"，这是行业简称：

- **三个9** = 99.9%
- **四个9** = 99.99%
- **五个9** = 99.999%

数字越多，要求越严，价格通常也越贵。

对于个人站长、小企业来说，**99.9%（三个9）** 已经足够用。真正需要"五个9"的，一般是银行核心系统、电商大促基础设施这个级别的业务。

### 3. 响应时间（Response Time）

出了问题，客服多久回你？这也是 SLA 的一部分。

企业级 SLA 通常会细分：

- P1（严重故障）：15 分钟内响应
- P2（主要影响）：2 小时内响应
- P3（轻微问题）：8 小时内响应

普通 VPS 厂商的 SLA 大多只约定工单响应时效，而不会细分故障等级。

### 4. 补偿机制（Remedies）

这是最容易被忽略、也最重要的一条。

没达到 SLA，服务商怎么赔？常见形式有：

- **服务积分**（Service Credit）：折抵下月账单
- **额外免费时长**：延长服务期
- **现金退款**：极少见，通常只在高端合同里

你需要看清楚：补偿的触发条件是什么？需要主动申请还是自动发放？上限是多少？

## SLA 和你有什么关系？

如果你只是个人博客，SLA 可能不是首要考虑因素。

但如果你在跑：

- 外贸独立站（订单丢失 = 真金白银损失）
- 程序 / API 服务（宕机 = 用户流失）
- 游戏服务器（掉线 = 玩家直接走人）
- 代理节点（稳定性是核心需求）

那 SLA 就是你选服务商必须扒开来看的东西，不能只看价格和配置。

## 怎么判断一家服务商的 SLA 值不值得信？

口头说"我们 99.9%"很容易，实际做到是另一回事。判断方法：

**1. 看补偿条款是否透明**

好的 SLA 会清楚写明：达不到多少给多少补偿。含糊写"视情况处理"的，等于没写。

**2. 看监控数据是否可查**

靠谱的服务商会提供状态页面（Status Page），让你实时看到历史可用性数据，而不是让你只听他们自己说。

**3. 看用户真实反馈**

上 V2EX、Reddit、Hostloc 这类社区搜一搜，看真实用户的使用体验，比官网的宣传可信得多。

**4. 看是否有退款保障**

有"X 天无理由退款"政策的厂商，通常对自己的服务更有信心。
